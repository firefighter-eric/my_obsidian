# 视频生成

## 页面状态

- 状态：正式 topic
- 事实基座：`wiki/summaries/` 优先

## 主题定义

本页讨论开放域视频生成与音视频联合生成主线，关注的问题是：视频模型如何从窄任务视频合成，走向可处理复杂动作、镜头语言、多模态参考、视频编辑与音画同步的通用创作引擎。

## 核心问题

- 如何从窄任务视频合成走向开放域、长脚本、多镜头的通用视频生成。
- 视频模型如何同时处理画面真实性、复杂动作稳定性、角色一致性与叙事镜头控制。
- 参考控制、视频编辑、续写与 extension 是否会成为比单轮 `text-to-video` 更重要的主线能力。
- 原生音频、音画同步与对白生成，是否正在把视频生成从“静音视觉模型”推进到音视频联合生成系统。

## 主线脉络 / 方法分层

- **早期窄域评测基座**：`What comprises a good talking-head video generation` 代表视频生成还主要按身份保持、口型同步、自然动作等局部维度来讨论，任务形态也更偏 talking-head 或单类视频合成。
- **world simulation / 物理一致性路线**：`Sora 2` 把视频生成继续解释为世界模拟能力的阶段性成果，重点放在物理合理性、跨镜头 world state 保持与角色注入。
- **cinematic realism + native audio 路线**：`Veo 3.1` 把 realism、physics、prompt adherence 与 native audio 整合到统一模型页中，代表 Google 将视频生成推进到带音频的 cinematic production 方向。
- **多模态参考 + 导演式控制路线**：`Kling VIDEO 3.0 Omni` 把 element、视频角色参考、语音绑定与 storyboard 提升为核心接口，代表更强的导演工作流取向。
- **reference-to-video 家族化路线**：`Wan2.6` 与 `Seedance 2.0` 都把 `R2V`、多镜头叙事、角色一致性与音画同步纳入主线。其中 `Wan2.6` 更强调家族升级，`Seedance 2.0` 更强调全维 benchmark 与 production readiness。
- **API 能力面路线**：`Vidu Q2-Pro` 说明视频生成的竞争不只发生在论文或 demo 层，还体现在 API 能力矩阵上，例如 `reference-to-video`、`video extension`、`motion sync` 与音频相关接口是否被正式暴露。

## 关键争论与分歧

- **视频生成的核心目标到底是“世界模拟”还是“内容生产工具”**：`Sora 2` 更偏前者叙事，而 `Kling / Seedance / Vidu / Wan` 明显更靠近 production workflow。
- **单轮 `text-to-video` 是否仍是中心任务**：现有新来源更支持“不是”。`reference-to-video`、编辑、续写、角色一致性与镜头规划已经成为主线。
- **音频是否只是附加功能**：从 `Sora 2`、`Veo 3.1`、`Kling 3.0 Omni`、`Wan2.6`、`Seedance 2.0` 的官方叙事看，native audio 与 audio-video alignment 已进入核心竞争维度。
- **当前视频生成是否已经稳定成熟**：现有来源普遍强调大幅进步，但 `Seedance 2.0` 仍在 extension 上暴露弱点，多数来源也缺少完全独立的第三方复核。
