# 视频生成

## 页面状态

- 状态：正式 topic
- 事实基座：`wiki/summaries/` 优先

## 主题定义

本页讨论开放域视频生成与音视频联合生成主线，关心的问题不是某个单一模型能否生成“看起来不错”的短视频，而是：**视频模型如何从窄任务合成器，演化为能处理复杂动作、多镜头叙事、角色一致性、参考控制、编辑续写与音画同步的创作系统。**

这个定义需要和相邻问题切开。它不等同于 talking-head synthesis，也不等同于单轮 `text-to-video`，更不等同于短视频特效工具大全。当前页面把 talking-head 视为早期评测基座，把 `text-to-video` 视为更大工作流中的一个入口任务，把原生音频、参考控制、续写和 extension 视为后续主线能力，而不是附加插件。

从当前 evidence base 看，本页最关键的主题判断是：**视频生成的中心竞争点已经从“能不能出视频”转向“能否稳定管理世界状态、角色一致性、镜头控制和音画联动”。** `Chen et al. 2020` 支撑早期局部评价维度，`Sora 2` 支撑 world simulation 叙事，`Veo 3.1` 支撑 native audio 与 cinematic realism，`Kling / Wan / Vidu / Seedance` 则共同支撑 production workflow、reference-to-video 与 API 能力面的崛起。

## 核心问题

- **视频生成如何从窄任务合成走向开放域与多镜头叙事**，而不在长程一致性和动作稳定性上崩溃。
- **视频模型的主线能力究竟是什么**：真实感、物理合理性、角色一致性、镜头语言，还是参考控制和编辑工作流。
- **单轮 `text-to-video` 是否仍是合适的主问题表述**，还是应该被更大的 reference / edit / continuation 工作流取代。
- **原生音频与音画同步是否已成为主线组成部分**，而不只是静音视频后的外挂能力。

## 主线脉络 / 方法分层

从当前 summary 组合出发，本页更适合按**视频系统面向的创作能力层**来分层，而不是按公司或发布时间逐条堆叠。

- **窄任务评测基座层**：`What comprises a good talking-head video generation` 代表视频生成早期仍主要围绕身份保持、口型同步、自然动作和视频质量等局部维度展开评测。这一层的重要性不在于 talking-head 是否代表整个视频生成，而在于它揭示了早期领域还没有统一的开放域目标，只能先从更窄、可测的子任务出发。
- **世界状态与物理一致性层**：`Sora 2` 把视频生成继续叙述为 `world simulation` 路线的一部分。官方反复强调复杂多镜头指令跟随、world state 持续性、物理合理性和角色注入能力。这条线的核心，不是某个镜头更像电影，而是模型是否开始具备**跨镜头维持一个持续世界**的能力。
- **cinematic realism 与原生音频层**：`Veo 3.1` 把 native audio、prompt adherence、visually realistic physics 与参考控制一起放进官方模型页，表明视频生成已经不再把音频当成外挂。这里的关键进展是：**视频系统开始被定义为音视频联合模型，而不是静音视觉模型加后期配音。**
- **导演式参考与多模态控制层**：`Kling VIDEO 3.0 Omni` 和 `Wan2.6` 共同表明，视频生成主线正在从纯 prompt 生成转向**reference-heavy workflow**。角色参考视频、语音绑定、element consistency、storyboard、多镜头脚本和 reference-to-video 都被提升为一等输入接口。换句话说，这一层解决的是“怎样像导演一样指定素材、角色和镜头”，而不是“怎样写更长的 prompt”。
- **平台化能力矩阵层**：`Vidu Q2-Pro` 很关键，因为它说明视频生成竞争不仅发生在论文页和 demo 页，也发生在**API 暴露出的能力边界**上。`reference-to-video`、`video extension`、`motion sync`、与音频相关的接口成为正式产品能力面。这意味着 topic 不能只看单个模型的视觉效果，还要看系统究竟向用户暴露了哪些可组合工作流。
- **统一多模态生产层**：`Seedance 2.0` 把 `text / image / audio / video` 四类输入统一到一个音视频生成框架中，并同时覆盖 `T2V / I2V / R2V`、编辑、continuation 和 extension。它的重要性不只是 benchmark 排名，而是进一步强化了这样一个判断：**视频生成正在从单任务模型，转向多模态输入、多任务输出的生产型系统。**

由此可以得到一个更强的综述判断：**视频生成主线已经不再以单轮 `text-to-video` 为唯一中心，而是逐步重组为“世界状态管理 + 多模态参考控制 + 音视频联合输出 + 工作流接口暴露”的复合系统。** 各家差异更多体现为强调哪一层更强，而不是是否仍停留在纯视觉短视频合成。

## 关键争论与分歧

- **视频生成的核心目标到底是世界模拟还是内容生产工具**：`Sora 2` 更明显沿 world simulation 叙事展开，而 `Kling / Wan / Vidu / Seedance` 明显更偏 production workflow 与素材控制。现有证据不支持简单判定哪条线“更正确”；更稳妥的说法是，两条叙事正在共同塑造视频生成，只是前者强调能力本体，后者强调可用接口。
- **单轮 `text-to-video` 是否仍是中心任务**：现有来源整体更支持“不是唯一中心”。`reference-to-video`、角色一致性、续写、extension、storyboard 和音视频绑定已经成为多家系统的显式能力面。因此更准确的写法是：`text-to-video` 仍是入口任务，但已经不足以代表视频生成主线。
- **音频是否只是附加功能**：从 `Sora 2`、`Veo 3.1`、`Kling 3.0 Omni`、`Wan2.6`、`Seedance 2.0` 的叙事看，答案越来越接近否定。native audio、对白、音效、环境声、音画同步都被放到核心宣传位，说明音频已经进入主线竞争维度。但当前多为官方来源，因此仍需保留“质量上限和稳定性尚待更多独立评测”的谨慎措辞。
- **当前视频生成是否已经成熟稳定**：现有证据只能支持“能力显著增强”，不能支持“已全面成熟”。原因有二：第一，绝大多数来源仍是官方模型页、用户指南或 model card；第二，`Seedance 2.0` 自身也暴露出 extension 等子任务上的短板。这意味着 topic 可以写“主线能力面快速扩张”，但不能写“所有关键子任务已收敛”。
- **官方产品页是否足以支撑强架构判断**：不能。当前页可以较稳地写产品能力层和工作流层主线，但关于训练数据、底层架构、统一模型机制的细粒度判断，必须持续保留不确定性标注。

## 未解决问题

- 当前页面已经能写出**窄任务评测、world simulation、native audio、导演式控制、平台能力矩阵、统一多模态生产**六层主线，但研究证据仍明显偏官方，论文级方法细节支撑不足。
- `Runway`、`Pika`、`Luma`、`MovieGen`、`Open-Sora`、`HunyuanVideo` 等重要路线尚未补入 `wiki/summaries/`，因此本页更像现阶段产业主线快照，而不是完整历史综述。
- 大多数现有来源仍不足以支撑关于训练数据、架构统一性、对齐方法和安全机制的强结论；因此本页当前更适合写能力面和工作流层判断，而非底层技术细节定论。
- world simulation、reference-to-video、extension、video editing、audio-video alignment 这几条线目前还缺 comparison 和 timeline 页面，导致本页虽然已有主线，但横向对照仍然不够稳固。
