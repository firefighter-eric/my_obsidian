# 扩散模型与文生图

## 页面状态

- 状态：正式 topic
- 事实基座：`wiki/summaries/` 优先

## 主题定义

本页讨论的是**以 diffusion / latent diffusion 及其后续图像生成路线为核心**的文生图研究与产品化主线。它不是“所有图像生成方法”的总页，也不是单纯的模型清单；其真正关心的问题是：图像生成为何从高成本研究原型，演化为可开放发布、可在消费级硬件运行、可承担编辑与生产工作流的基础模型体系。

这个主题的边界需要明确。与 `传统 CV` 相比，本页聚焦的是**生成接口而非识别接口**。与 `Qwen 系列` 相比，本页不是按公司或模型家族来写，而是按图像生成问题如何分层来组织。与 [图像分层 layered](./%E5%9B%BE%E5%83%8F%E5%88%86%E5%B1%82%20layered.md) 相比，本页只把 layered 看成文生图主线向“原生可编辑中间表示”延伸的一条支线，而不在这里展开 layered 内部谱系。

从当前证据看，本页最重要的判断是：**文生图主线已经不再等于“给一句 prompt 生成一张图”**。`LDM` 支撑的是低成本高保真生成底座，`Stable Diffusion` 支撑的是开放权重和社区生态，`Qwen-Image` 支撑的是复杂文字渲染与编辑能力，`AlphaVAE / Qwen-Image-Layered` 则支撑了“可编辑图像表示”正在从边缘需求转成独立问题设定。

## 核心问题

- **扩散模型为什么能成为图像生成的主线基础设施**，而不是只作为一类高质量但高成本的研究模型存在。
- **图像生成的竞争维度究竟是什么**：是视觉质量、算力成本、开放发布，还是文字渲染、编辑一致性与工作流控制。
- **生成模型是否正在从 RGB 输出器转向可编辑中间表示系统**，例如透明图像、`RGBA` 表示、图层分解。
- **“文生图”是否仍是足够准确的主题名**，还是应该理解为“生成 + 编辑 + 结构控制”的更大问题域。

## 主线脉络 / 方法分层

从当前 summary 组合看，本页更适合按**模型在工作流里承担的角色**来分层，而不是按时间先后简单排队。

- **潜空间生成底座层**：`High-Resolution Image Synthesis with Latent Diffusion Models` 是这一主题的真正技术起点之一。它的决定性意义不是提出又一个 diffusion 变体，而是把扩散过程转移到潜空间中，使高保真图像生成第一次兼顾了**可接受的训练与推理成本**。同时，cross-attention 让文本、框、掩码等条件接口可以进入统一生成器。这一层解决的是“生成是否足够便宜且足够可控，能够成为平台能力”的问题。
- **开放基础设施层**：`Stable Diffusion Launch Announcement` 不是技术论文，但在 topic 层的地位非常高，因为它标志着 latent diffusion 从研究线进入**开放权重、消费级 GPU、社区协作**的公共基础设施阶段。这里的核心突破不只是质量，而是研究路线首次获得了广泛可复用性。换句话说，`LDM` 解决“能不能做”，`Stable Diffusion` 解决“谁都能不能拿来做”。
- **生产控制层**：`Qwen-Image Crafting with Native Text Rendering` 显示，后续图像生成模型的竞争点已经从单纯的 photorealism 外移到**复杂文字渲染、双语排版、图像编辑和设计任务适配**。这条线说明后续模型不再只服务于艺术化 prompt，而是在向海报、PPT、电商图、版式图等高文本密度工作流靠近。当前页面中，`Qwen-Image` 是这一层最清晰的代表。
- **可编辑表示层**：`AlphaVAE` 与 `Qwen-Image-Layered` 共同表明，图像生成主线正在向**可编辑中间表示**延伸。`AlphaVAE` 的关键意义是把 `RGBA` 作为独立表征对象和评测对象固定下来；`Qwen-Image-Layered` 的关键意义则是把单张 RGB 图像分解成多层 `RGBA` 直接做成模型能力。前者解决底层表示和 benchmark，后者解决图层分解与 inherent editability。二者并不等价，但共同支撑了“生成模型开始内生支持编辑工作流”这一更高层判断。

把这四层连起来，当前可以形成一个更强的综述主线：**文生图的长期演化不是单向追求更好看的图，而是不断把成本、开放性、可控性和可编辑性纳入同一个基础模型体系。** `LDM` 让扩散生成可扩张，`Stable Diffusion` 让其成为开放生态，`Qwen-Image` 让其面向高文本密度和编辑任务，`AlphaVAE / Qwen-Image-Layered` 则把“图像本体是什么”从 RGB 成品推向 layered / alpha-aware 表示。

## 关键争论与分歧

- **文生图主线的决定性分水岭到底是技术突破还是开放发布**：现有证据显示，两者不能互相替代。没有 `LDM`，就没有低成本高保真底座；没有 `Stable Diffusion` 式开放发布，文生图也不会形成后来那种广泛社区基础设施。因此更准确的说法是：技术路线与开放生态共同塑造了主线，而不是谁完全支配谁。
- **后续图像生成的核心能力是否已经从“生成”转向“编辑”**：当前证据较强地支持“至少在产品化竞争中，是这样”。`Qwen-Image` 明确把 editing 和 text rendering 置于核心卖点，`Qwen-Image-Layered` 则直接把可编辑表示当成问题本体。但现有证据仍不足以说明单轮 `text-to-image` 已经失去中心地位；更稳妥的判断是，**生成已不再足够，编辑成为必须补齐的主线能力**。
- **文本渲染是否只是 niche benchmark**：当前不支持这种轻描淡写。`Qwen-Image` 把长文本、双语排版、海报/PPT 场景作为显式能力面，说明文字渲染已经从“很难但边缘”的 demo，转向生产工作流中高度实际的评价维度。
- **`RGBA / layered` 是否只是设计软件特例**：现有来源更支持“不是”。`AlphaVAE` 从表示学习和 benchmark 层面独立化 `RGBA`，`Qwen-Image-Layered` 从模型能力层面独立化图层分解。这意味着 layered 已经不是纯 UI 特性，而是生成模型内部表示问题的一部分。但这个判断仍受限于当前来源数量，尚不能推出它会成为所有图像模型的统一标准接口。
- **本页是否还能统一称为“扩散模型与文生图”**：当前可以，但要带约束地理解。`LDM` 明确是 latent diffusion；`Qwen-Image` 明确为 `MMDiT` 图像基础模型；`Qwen-Image-Layered` 是在预训练生成模型上迁移出的分解器。本页使用这个题名，是为了组织研究主线，而不是断言所有节点公开架构完全同构。

## 未解决问题

- 当前页面已经能较稳地写出**潜空间底座、开放生态、文字渲染/编辑、可编辑表示**四层主线，但 `DALL·E 2/3`、`Imagen`、`ControlNet`、`SDXL` 等关键节点尚未补入 `wiki/summaries/`，因此整条文生图历史仍明显不完整。
- `FLUX.2` 目前主要由官方概览页支撑，其在架构层、训练层和 benchmark 层的判断证据仍弱于 `LDM` 或 `Qwen-Image-Layered`。因此本页可以把它写成生产级控制能力代表，但不宜写成稳定技术分水岭。
- `Qwen-Image` 目前仍主要依赖官方博客，关于训练数据、对齐策略与具体开放范围的结论必须继续保留谨慎措辞。
- `RGBA / layered` 虽已形成可写主线，但其前史仍残缺，`LayerDiffuse`、`DreamLayer`、`Text2Layer`、`LayerD` 等节点尚未补齐 summary；因此本页只能确认“它已成为独立问题域”，还不能完整重建其演化谱系。
