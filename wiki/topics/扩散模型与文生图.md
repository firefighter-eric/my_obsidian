# 扩散模型与文生图

## 页面状态

- 状态：正式 topic
- 事实基座：`wiki/summaries/` 优先

## 主题定义

本页讨论以 diffusion / latent diffusion 及其后续图像生成路线为核心的文生图研究与产品化主线。它关注的问题不是“所有视觉生成技术”，而是：图像生成如何从高成本研究模型演化为开放权重、可在消费级硬件上运行、可做多参考编辑、可做文字渲染与生产工作流控制的基础模型体系。与 `传统CV` 相比，本页聚焦生成而非识别；与 `Qwen 系列` 相比，本页按图像生成方法线组织，而不是按单一家族组织。

## 核心问题

- 扩散模型如何在质量、算力成本与控制能力之间取得平衡。
- 文生图模型为何会从“像素空间高成本生成”转向潜空间、再转向更强的控制与工作流接口。
- 开放权重与社区生态为何会成为图像生成主线的重要分水岭。
- 文本渲染、图像编辑、多参考控制为何会成为后续文生图模型的竞争焦点。

## 主线脉络 / 方法分层

- latent diffusion 奠基：`High-Resolution Image Synthesis with Latent Diffusion Models` 把扩散过程从像素空间转入潜空间，并通过 cross-attention 接入文本等条件输入。这条线的意义在于，它首次把“高质量扩散生成”与“可接受的算力成本”结合起来。
- Stable Diffusion 的开放扩张：`Stable Diffusion Launch Announcement` 说明 latent diffusion 很快被工程化为开放 text-to-image 模型。这里的核心转变不是单纯性能刷新，而是“消费级 GPU 可跑 + 开放权重 + 社区生态”的公共基础设施化。
- 生产级控制深化：`FLUX.2 Overview` 代表 post-Stable-Diffusion 时代的一个重要方向，即把多参考编辑、精确颜色控制、结构化 prompting、文本渲染和生产工作流集成到一个分层模型家族中。这里的主线不再只是从 prompt 出图，而是让图像生成进入真实设计、广告、电商与 UI 流程。
- 原生文本渲染与编辑一体化：`Qwen-Image Crafting with Native Text Rendering` 代表另一条更强调“文字可读性”和双语渲染的路线。它说明后续图像生成模型的竞争，不只在 photorealism，也在复杂版式、长文本、海报/PPT 等高文本密度场景。
- `RGBA` 与图层表示支线：`AlphaVAE` 与 `Qwen-Image-Layered` 共同表明，文生图主线正在向透明图像、图层分解和原生可编辑表示延伸。前者补的是 `RGBA` VAE 与 benchmark，后者补的是从单张 RGB 图像直接进入多层 `RGBA` 工作流的分解接口。

## 关键争论与分歧

- 文生图的核心突破到底是开放性还是模型质量：Stable Diffusion 的历史地位主要来自开放发布与硬件可达性，而 FLUX.2、Qwen-Image 则更强调生产质量与控制。这两类价值并不相同。
- 图像生成是否已从“生成”转为“编辑 + 工作流”：从 FLUX.2 与 Qwen-Image 的来源看，当前更稳妥的判断是，单轮 text-to-image 已不再足够，编辑、一致性和结构化控制已成为主线竞争点。
- 文本渲染是不是边缘能力：较早文生图系统通常把文字视为薄弱环节，但 Qwen-Image 与 FLUX.2 都把 text rendering 作为显式卖点，说明它已从边缘 demo 变成核心评价维度。
- `RGBA` / layered image 是否只是 niche 工作流：现有两篇新来源更支持“不是纯边缘需求”。`AlphaVAE` 把透明图像评测和潜表示独立化，`Qwen-Image-Layered` 则把图层分解直接做成模型能力，说明这一方向正在从设计软件特例转成生成模型中的独立问题设定。
- 当前所有前沿家族是否仍可统称 `diffusion` 模型：Stable Diffusion 的 latent diffusion 身份明确；Qwen-Image 明确是 `MMDiT`；FLUX.2 在当前来源中更强调产品能力而非完整架构披露。因此，把它们统一放入“扩散模型与文生图”主题是为了研究线组织，而不是断言三者公开架构完全同构。

## 未解决问题

- 当前知识库尚未补入更完整的文生图时间线，例如 Imagen、DALL·E 2/3、SDXL、ControlNet 等关键节点，因此方法分叉仍偏粗。
- FLUX.2 当前仅有官方文档级来源，缺少更完整的研究论文或独立技术报告支撑架构层判断。
- Qwen-Image 目前仍以官方博客为主，关于训练数据、对齐策略和开源范围的细节还不足。
- 当前库里虽已补入 `AlphaVAE` 与 `Qwen-Image-Layered`，但 `LayerDiffuse`、`DreamLayer`、`Text2Layer`、`LayerD` 等前序路线仍未系统补齐，因此 `RGBA / layered image` 支线的演进脉络还不完整。
