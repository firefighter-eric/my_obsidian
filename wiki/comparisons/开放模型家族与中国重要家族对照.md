# 开放模型家族与中国重要家族对照

## 比较目标

本页比较当前知识库中一批重要的开放模型家族，以及两个不能缺席的中国重要家族对照节点 `GLM` 与 `Kimi`。重点不是做排行榜，而是澄清它们在开放性、家族角色、技术主轴与知识组织方式上的差异。

## 核心结论

- 开放模型主线已经明显多极化，不再只有 `Llama` 一条线；`Mistral`、`Gemma`、`DBRX`、`OLMo 2`、`GLM` 等都在形成不同风格的家族。
- `open-weight`、`开放发布但限制较多`、`API 或闭源` 必须明确区分，否则会把家族性质写错。
- 中国重要家族节点不能只看 `Qwen`；`DeepSeek` 与 `GLM` 是需要保留的开放 / 半开放主线，`Kimi` 则是高影响但非 open-weight 的对照节点。
- `StarCoder2` 这类代码家族和 `Phi-3 / OpenELM` 这类高能力密度小模型，说明开放模型竞争已从“谁更大”转向“谁更适合特定部署与工作流”。

## 对照维度

### 1. 家族与组织

- `Meta`：`LLaMA / Llama 2 / Llama 3` 代表开放大模型主干。
- `Mistral AI`：`Mistral 7B / Mixtral` 代表高效 dense 与开放 `MoE` 路线。
- `Google / DeepMind`：`Gemma / Gemma 2 / Gemma 3` 代表大厂开放小中尺寸家族。
- `Databricks / MosaicML`：`MPT / DBRX` 代表平台公司主导的开放模型与工程生态路线。
- `Ai2`：`OLMo 2` 代表比一般 open-weight 更强调 fully open 的研究路线。
- `THUDM / Zhipu`：`GLM` 代表中文重要家族与双语开放主线。
- `DeepSeek`：`DeepSeek-V3 / DeepSeek-V4` 代表高效 MoE、长上下文与 agent-oriented 开放模型路线，`DeepSeek-R1` 则连接 reasoning RL。
- `Moonshot AI`：`Kimi` 代表高影响但非 open-weight 的中国重要对照家族。

### 2. 开放性

- `open-weight`：`BLOOM`、`MPT`、`Mistral 7B`、`Mixtral`、`Gemma`、`Gemma 2`、`StarCoder2`、`DBRX`、`OpenELM`、`Phi-3`、`GLM-130B`、`DeepSeek-V3`、`DeepSeek-V4`。
- `开放发布但限制较多`：`Falcon 3`。
- `更接近 fully open research release`：`OLMo 2`。
- `API 或闭源`：`Kimi`。

### 3. 技术主轴

- 超大开放协作：`BLOOM`。
- 工程化商用友好：`MPT`。
- 高效 dense：`Mistral 7B`。
- 开放 `MoE`：`Mixtral`、`DBRX`、`DeepSeek-V3 / DeepSeek-V4`。
- 大厂轻量开放家族：`Gemma`、`Gemma 2`。
- 代码家族：`StarCoder2`。
- 端侧效率与小模型：`OpenELM`、`Phi-3`。
- fully open 研究透明：`OLMo 2`。
- 中文重要家族：`GLM`。
- 长上下文与 agent-oriented 开放模型：`DeepSeek-V4`。
- 中国高影响非 open-weight 对照：`Kimi`。

### 4. 在当前知识库中的组织方式

- `BLOOM / Mistral / Gemma / DBRX / OpenELM / Phi-3 / OLMo 2` 应归入 `LLM 预训练` 的开放模型家族主线。
- `GLM` 同样进入 `LLM 预训练`，但在叙述中要明确它属于中文重要家族节点，而不只是一般国际开放模型补充。
- `DeepSeek-V4` 应进入 `LLM 预训练` 的开放 MoE 与长上下文效率主线，但其 reasoning mode、agent benchmark 与工具调用设计应拆到 `LLM RL` 或后续 agent topic 中处理。
- `Kimi` 也应出现在 `LLM 预训练` 和必要的 `LLM RL` 语境里，但必须保留“不是 open-weight”的标签。
- `StarCoder2` 仍放在 `LLM 预训练` 主线中，因为它代表开放代码模型家族，而不是孤立工具模型。

## 关键分歧

- “开放模型”不能等同于“任何公开可用的大模型”；`Kimi` 这样的 API 型家族需要明确剥离。
- “开源”也不能只看是否能下载权重；`OLMo 2` 与一般 open-weight 家族在开放程度上有实质差别。
- 中国模型主线不能只保留一个 `Qwen`；至少还需要 `DeepSeek` 与 `GLM` 作为开放 / 半开放家族对照，以及 `Kimi` 作为高影响闭源家族对照。
